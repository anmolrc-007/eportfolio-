HyperText Markup Language

HTML — The Skeleton

What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. It uses tags wrapped in angle brackets to describe the structure and meaning of content. Every website begins with HTML.

Tags & Elements

HTML uses opening and closing tags. An element is everything from the opening tag to the closing tag: <p>Hello</p>. Some tags are self-closing like <img />.

Attributes

Attributes provide extra information about elements. They appear inside the opening tag: <a href="url">. Common attributes include class, id, src, href, alt.

Semantic HTML

Semantic elements describe their own meaning. <header>, <nav>, <main>, <article>, <footer> all tell browsers (and search engines) what each section is for.

Accessibility

Good HTML is accessible. Using correct semantic elements, alt text on images, and proper heading hierarchy (h1→h6) ensures screen readers and assistive tech can interpret the page correctly.

Document Structure

Every HTML document follows a standard skeleton. Understanding this structure is the foundation of all web development.
HTML — Standard Document Structure 
<!-- Tells the browser this is HTML5 -->
<!DOCTYPE html>

<html lang="en">

  <head>
    <!-- Not visible to the user; metadata lives here -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width">
    <title>Page Title</title>
    <link rel="stylesheet" href="../css/html.css">
  </head>

  <body>
    <!-- All visible content goes here -->
    <header>...</header>  <!-- Site header / nav -->
    <main>...</main>    <!-- Primary content -->
    <footer>...</footer>  <!-- Footer -->
    <script src="../js/app.js"></script>
  </body>

</html>

DOM Tree

Document Object Model (DOM)
<html>
<head>
<title> My Page </title>
<body>
<header>
<h1> Welcome
<main>
<p> Content here
<img src="pic.jpg" />

The DOM is the browser's representation of the HTML as a tree of objects. JavaScript uses the DOM to read and change page content dynamically.

Essential HTML Elements

Headings

Six levels: <h1> to <h6>. Only one <h1> per page. Headings create hierarchy and are essential for SEO and accessibility.

Text Elements

<p> paragraph, <span> inline, <strong> bold/important, <em> italic/emphasis, <br> line break.

Lists

<ul> unordered (bullets), <ol> ordered (numbers), <li> list item. Lists can be nested inside each other for hierarchical content.

Links & Images

<a href="url"> creates hyperlinks. <img src="path" alt="desc"> embeds images. The alt attribute is required for accessibility.

Div & Section

<div> is a generic block container. <section> groups related content with meaning. Use semantic elements over generic divs when possible.

Tables

<table>, <thead>, <tbody>, <tr> (row), <th> (header cell), <td> (data cell). Use for tabular data only, not layout.

Forms & Tables

HTML — Contact Form 
<form action="/submit" method="POST">

  <label for="name">Your Name:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="role">Role:</label>
  <select id="role" name="role">
    <option value="dev">Developer</option>
    <option value="design">Designer</option>
  </select>

  <button type="submit">Submit</button>
</form>
Forms are the primary way users send data to a web application. The action attribute specifies where the data goes; method specifies how (GET for fetching, POST for sending sensitive data).
