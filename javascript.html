JavaScript Fundamentals

JavaScript — The Interaction

Why JavaScript?

JavaScript adds behaviour to web pages. It can change HTML content, respond to user input, animate elements, and communicate with servers — all inside the browser.

Dynamic Pages

JS can modify text, attributes, styles, and page structure after the HTML has loaded. This makes websites interactive instead of static.

Event Driven

JavaScript reacts to clicks, keyboard input, mouse movement, form submissions, and more. Events are how users interact with the page.

Browser APIs

Built-in browser APIs let JS work with the DOM, storage, network requests, audio, video, and even device sensors.

Application Logic

JS handles conditions, loops, functions, data structures, and calculations — making it the main language for front-end application logic.

Document Object Model (DOM)

The DOM is the browser's representation of HTML. JavaScript can select elements, read their content, update them, and attach event listeners to them.

Selecting Elements

Use querySelector and querySelectorAll to find elements. IDs are unique, while classes can match multiple elements.

Updating Content

Change text with textContent or HTML with innerHTML. Append, replace, or remove elements to update page structure dynamically.

Event Listeners

Use addEventListener('click', ...) to run code when users interact. Events are the core of interactive web experiences.

Live Updates

JS can react instantly to user actions, like live filtering a list, updating totals, or validating forms before submission.

Events & Interactions

Mouse & Keyboard Events

JavaScript can respond to many events — click, mousemove, keydown, submit, scroll, focus, and more.

Events allow JS to make pages feel alive. The browser sends an event, and your script decides how the page should respond.
